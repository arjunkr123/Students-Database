Coding Conventions

Naming

PHP

  • Class names start with a capital letter, then small letters or digits. Each word starts with a capital letter.
    Example: DatabaseSystem
    Exception: classes for a page or a service, which must repect the name of the URL, meaning only small letters, and underscores to separate words.
  • Properties are in small letters, each word is spearated by an underscore.
    Example: user_id
  • Methods start with a small letter, then each word starts with a capital letter.
    Example: getUserId

JavaScript

  • Files use only small letters, each word is separated by an underscore.
    Example: context_menu.js
  • For classes, there are two possibilities:
    • When a file is just providing a class, to make it clearer the class can have the name of the file, meaning it uses only small letters and words are separated by an underscore.
    • Any other class should follow same way as PHP classes: start with a capital letter, then small letters or digits. Each word starts with a capital letter.
  • Properties or variables are using small letters, each word is separated by an underscore.
    Example: user_id
  • Methods or functions start with a small letter, then each word starts with a capital letter.
    Example: getUserId

Database

  • Tables are following classes conventions: names start with a capital letter, then small letters or digits. Each word starts with a capital letter.
    Example: UserRights
  • Columns are following properties conventions: in small letters, each word is spearated by an underscore.
    Example: first_name
